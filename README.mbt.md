# LunaLab

LunaLab 是一个基于 MoonBit + Qt 的原生 AI 聊天客户端。

它的重点不是把模型接口包上一层桌面壳，而是把对话本身做成一种可组织、可回看、可分叉的工作对象。相比常见的线性聊天记录，LunaLab 更关心“这段对话是怎么长出来的”“我现在正站在哪个历史节点上继续工作”。

## 核心特色

- 对话树而不是线性历史。消息会形成树状结构，而不是只能从上到下单线追加。
- 可以从任意历史节点继续。当前工作位置由 `HEAD` 表示，切换 `HEAD` 后就能沿另一条路径继续推进对话。
- 历史编辑不以覆盖为目标。底层模型为“编辑并分叉”保留了语义基础，旧路径可以继续保留。
- 支持将长链路压缩为摘要节点，用更短的上下文表达一段已经完成的历史。

这些设计的目标很简单：把 AI 对话从一次性消费的聊天窗口，变成可以反复整理和推进的工作流。

## 当前状态

项目目前已经可以作为原生桌面应用运行，包含聊天页、树视图、本地状态持久化，以及基于 OpenAI-compatible 接口的模型请求能力。

现阶段更偏向“核心交互和数据模型成型”的阶段，重点已经放在对话树语义、状态管理和桌面端交互上。

## 运行要求

- MoonBit 工具链
- `clang`
- Qt 运行环境
- [`KeqingMoe/qt`](https://github.com/KeqingMoe/qt) 对应的本地构建条件

项目默认使用 `native` 目标运行。

## 快速开始

直接启动应用：

```bash
moon run
```

## 开发

常用命令：

```bash
moon check
moon test
moon info
moon fmt
```

推荐在提交前至少执行：

```bash
moon info
moon fmt
moon test
```

## 项目结构

```text
.

├── ui/        # Qt 界面、状态管理、持久化、网络请求
├── lunalab.mbt
├── moon.mod.json
└── moon.pkg
```
